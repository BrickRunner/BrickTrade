# 🔄 Переключение в режим торговли

## ⚠️ ВАЖНО: Прочитайте внимательно перед включением торговли!

Текущий бот настроен в **БЕЗОПАСНОМ режиме мониторинга**. Для переключения в режим торговли потребуется несколько шагов.

## 🛡️ Текущий режим (MONITORING)

```env
ARB_MONITORING_ONLY=true    # Только мониторинг
ARB_MOCK_MODE=false          # Реальные API
ARB_DRY_RUN_MODE=true        # Без сделок
```

**Что работает:**
- ✅ Мониторинг 80+ пар на OKX и HTX
- ✅ Реальные цены в реальном времени
- ✅ Уведомления о возможностях
- ❌ Торговля отключена

## 🔓 Режим DRY RUN (симуляция сделок)

### Шаг 1: Измените .env

```env
ARB_MONITORING_ONLY=false   # Разрешить торговые функции
ARB_MOCK_MODE=false          # Реальные API
ARB_DRY_RUN_MODE=true        # Симуляция (безопасно)
```

### Шаг 2: Добавьте HTX API ключи

```env
# HTX API (нужны для торговли)
BYBIT_API_KEY=ваш_ключ
BYBIT_SECRET=ваш_секрет
```

Получить: https://www.bybit.com/app/user/api-management

**Права:** Read + Write (НЕ давайте права на вывод средств!)

### Шаг 3: Настройте параметры

```env
# Консервативные настройки
POSITION_SIZE=0.01           # Малый размер позиции
ENTRY_THRESHOLD=10.0         # Высокий порог входа (10%)
EXIT_THRESHOLD=5.0           # Высокий порог выхода (5%)
MAX_RISK_PER_TRADE=0.01      # 1% от депозита
```

### Шаг 4: Перезапустите бота

```bash
# Остановите текущий бот (Ctrl+C)
python main.py
```

**Что изменится:**
- ✅ Симуляция входа/выхода из сделок
- ✅ Уведомления о P&L (не реальные)
- ✅ Проверка логики торговли
- ❌ Реальные ордера НЕ размещаются

## 🔍 Что нужно изменить в коде

### Если нужен UI для торговли

Замените `arbitrage_handlers_simple.py` на полную версию `arbitrage_handlers.py`:

```python
# В main.py
# Было:
from handlers import arbitrage_handlers_simple as arbitrage_handlers

# Станет:
from handlers import arbitrage_handlers
```

Это вернет:
- 🎯 Стратегии (5 видов арбитража)
- 📊 Торговые параметры
- 📈 Статистику сделок
- 🎛 Расширенные настройки

### Добавьте торговые обработчики

Раскомментируйте в `main.py`:

```python
# Стратегии (5 стратегий)
dp.callback_query.register(arbitrage_handlers.cb_arb_strategies_menu, lambda c: c.data == "arb_strategies_menu")
dp.callback_query.register(arbitrage_handlers.cb_arb_toggle_strategy, lambda c: c.data.startswith("arb_toggle_strategy:"))
dp.callback_query.register(arbitrage_handlers.cb_arb_strategies_start, lambda c: c.data == "arb_strategies_start")
dp.callback_query.register(arbitrage_handlers.cb_arb_strategies_stop, lambda c: c.data == "arb_strategies_stop")
```

## ⚡ Риски

### Технические риски
- Сбои API бирж
- Проблемы с интернетом
- Баги в коде
- Изменения в API бирж

### Торговые риски
- Slippage при исполнении
- Комиссии съедают прибыль
- Funding rates против вас
- Резкие движения рынка
- Ликвидность исчезает

### Финансовые риски
- Частичное исполнение ордеров
- Застревание в позиции
- Невозможность выйти
- Потеря депозита
