## 1. РЕЖИМЫ РАБОТЫ СИСТЕМЫ

Система поддерживает 4 режима работы:

| Режим | monitoring_only | mock_mode | dry_run_mode | Описание |
|-------|-----------------|------------|---------------|----------|
| **MOCK** | false | **true** | false | Симуляция, без реальных API |
| **DRY_RUN** | false | false | **true** | Реальные данные, без ордеров |
| **MONITORING** | **true** | - | - | Только мониторинг, без торговли |
| **REAL** | false | false | false | Реальная торговля |

**Текущий статус из логов:** Бот работал в MOCK режиме (видно из логов `[MOCK]`).

## 6. ЧЕК-ЛИСТ ПЕРЕД ТЕСТОВЫМ ЗАПУСКОМ (10$)

### Перед запуском:

- [ ] **Убедиться что установлено:** `ARB_MONITORING_ONLY=true` или `ARB_DRY_RUN_MODE=true`
- [ ] **Проверить API ключи OKX:**
  - Режим: Single-currency margin (не Simple!)
  - Права: Read + Trade
- [ ] **Проверить баланс:** Минимум $10 на OKX
- [ ] **Установить минимальный размер позиции:** `POSITION_SIZE=0.001` (~$5 при BTC=$50000)
- [ ] **Настроить лимиты риска:**
  ```
  MAX_POSITION_PCT=0.10  # 10% от депозита max
  MAX_DELTA_PERCENT=0.01  # 1% max перекос
  EMERGENCY_MARGIN_RATIO=0.2  # Экстренное закрытие при 20% марже
  ```

### Во время работы мониторить:

- [ ] Логи на наличие ошибок API
- [ ] Баланс позиций (delta не должен превышать 1%)
- [ ] PnL сделки (не должен быть отрицательным при низкой волатильности)
